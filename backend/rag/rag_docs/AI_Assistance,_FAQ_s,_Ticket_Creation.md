TITLE: AI Assistance, FAQ s, Ticket Creation

CATEGORY: IT
PRIORITY: MEDIUM

CONTENT:
This document explains how users can search FAQs, create support tickets, and interact with the AI assistant in the Smart Service Desk system.

For Common Users (Customers):

Searching FAQs:
Users can search frequently asked questions (FAQs) from the home page without logging in. FAQs cover common issues such as login problems, password resets, access errors, and general system usage. Users should enter relevant keywords in the FAQ search bar to quickly find answers.

Using AI Assistance:
The AI assistant provides instant, context-aware responses based on the knowledge base and uploaded documents. Users can ask questions in natural language, such as:
- "How do I reset my password?"
- "Why am I unable to log in?"
- "How do I raise a support ticket?"

If the AI finds a relevant answer in the knowledge base, it will respond immediately. If the AI is not confident, it will suggest contacting a support agent.

Creating Support Tickets:
If users cannot find a solution through FAQs or AI assistance, they can create a support ticket by clicking the "Contact Agent" or "Create Ticket" button.

To create a ticket, users must:
- Log in or register
- Provide a ticket subject
- Describe the issue clearly
- Submit the ticket

Once submitted, the ticket is assigned a unique ticket ID and its status is set to OPEN. Users can view their ticket status and conversation history from the Ticket Management page.

For Agents and Employees:

Handling Tickets:
Support agents can view assigned tickets, read user messages, and respond through the agent dashboard. Agents should provide clear and actionable responses and update the ticket status as required.

Escalation:
If an issue requires higher-level support, agents can escalate the ticket to the appropriate team.

Ticket Resolution:
After resolving the issue, agents should close the ticket. Closed tickets remain visible for reference but cannot be replied to.

System Behavior:
The AI assistant does not automatically create tickets. Ticket creation always requires explicit user confirmation. The system ensures a seamless transition from AI assistance to human support when needed.
